SECTION 11 - Update the Aims class

Question: What happens when you don't update Aims class to catch PlayerException?

Answer:
Since PlayerException extends Exception, it is a CHECKED exception in Java.
Java's compiler enforces that every checked exception must be either:
  (1) caught using a try-catch block, or
  (2) declared in the method signature with "throws PlayerException".

If the Aims class is not updated and we simply call play() without any handling,
the compiler will refuse to compile the code and report a compile-time error:
  "unreported exception PlayerException; must be caught or declared to be thrown"

In contrast, if PlayerException extended RuntimeException instead, it would be an
unchecked exception and the compiler would not enforce handling - the program would
compile but could crash at runtime with an unhandled exception stack trace.


SECTION 14 - Exception Hierarchical Tree

All self-defined exceptions in AimsProject and their inheritance hierarchy:

  java.lang.Exception
  |
  +-- PlayerException       (package: hust.soict.globalict.aims.exception)
  |     Thrown when a media object cannot be played
  |     (e.g., DigitalVideoDisc, Track, or CompactDisc with length <= 0)
  |
  +-- LimitExceededException  (package: hust.soict.globalict.aims.exception)
        Thrown when the cart already has MAX_NUMBERS_ORDERED (20) items
        and the user tries to add one more.

Both exceptions are checked exceptions (they extend Exception directly),
so callers are required by the compiler to handle or declare them.
