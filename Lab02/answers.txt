[Section 15] Passing Parameters
- Question 1: After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two objects still remain?
- Answer 1: Because Java is strictly Pass-by-Value. For objects, Java passes a copy of the reference. The swap() method only swaps these local copies of the references inside the method, leaving the original references in the main method unchanged.

- Question 2: After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is the title of the JungleDVD changed?
- Answer 2: Because the copied reference still points to the same original object in the heap memory. When we call setTitle() using this copied reference, it accesses and directly modifies the internal data of that original object.
